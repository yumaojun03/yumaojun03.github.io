关于我

历程

因为喜欢而走上编程这条路，曾经干过设备维护, 机遇巧合让我接触到了Linux, 自此开启了我1年多的Linux自学之旅, 但是想要真正成为一名Linux运维工程师 还是不够, 我选择参加了马哥网络班的培训, 这算是我人生的另外一个转折点: 遇见马哥, 他是一个标杆, 一个精神领袖, 让我不甘平凡, 燃起斗志. 最终转行成为一个真正的Linux运维人员。

我做起了Linux运维，而此时Python正火, 并且运维过程中好多自动化的工具也需要使用Python来开发, 在有了Shell编程经验过后, 我开始学习Python, 一门真正图灵完备的语言, 虽然大学也学过C,但是没排上用场过, 真正让我感受到通过编程来解决问题的力量的是Python。

后来虚拟化掀起了运维的另外一场变革, 从最开始的Xen KVM Exsi到最后的IaaS平台, 到最后OpenStack的盛行, 突然觉得Python不仅仅可以用来写一些小工具和Web, 而且可以实现如此庞大的一个系统, 于是开始学习OpenStack不仅是编码, 主要是它的系统架构设计, 当然也期望有一天能参与到这个庞大的工程中去。

在工作中也会与OpenStack也相关,但是最终还是没有真正进入OpenStack社区, 而且也意识到OpenStack的一些痛点, 以及使用Python开发时的有些不足, 后面Docker的流行, 一些轻量级的平台可以使Docker快速开始构建, 于是开始深入了解Docker, 并且开始学习Golang, 也基于Swarm做过一个公司的Demo项目, 遗憾的是最后依然没有继续。

最终造化弄人, 最终在做IOT平台时, 前期参考OpenStack架构, 以它方式来开发组件(Python), 最后由于某些组件高性能的要求, 换成使用Golang来开发, 最后都重构成了Golang, 没想到我对Openstack和Docker的积累 会以这种方式被使用到。

想要自己能写一个完整的应用出来, 光Pythoh/Golang是不够的, 所以JavaScript, HTML, CSS 都得使用, 最后不得不评价下js, 这真是一门万能的语言, 就是坑多。

因为一定的缘分参与的开源堡垒机项目：JumpServer，现在依然是代码的核心贡献者之一。

博客

写博客的三种境界基本都经历了。

51CTO

最开始在51CTO上写博客，刚开始写得很慢，并且可读性也不好，而且写的方式也是采用, 先在未知笔记上写，写完同步到51CTO这种方式，在排版上也经常会出一些问题。

WordPress

后来心血来潮，买了一台虚拟主机，用WordPress搭建了一个个人博客，然后用WordPress提供的编辑器写博客，这样每年都得续费，维护虚拟主机也会耗一部分精力。这个博客已经有点时间没更新了，地址: http://www.yumaojun.com。

我担心格式问题，所以并未打算迁移。

Github Page

不想再维护虚拟主机了，vps又实在太贵，而且本身对github又比较熟，所以理所应当的选择了Github Pages服务，来搭建静态博客。对于写博客工具，我之前使用Pelican，这是一个python的工具，因为对python比较熟，后来发现我的主要问题还是得搞定模板(js)，所以最后还是放弃了pelican，改用Hexo，这个用JS写的工具来写博客。

还有markdown写起来真心不错，我用vscode写代码，同时vscode对markdown也支持的很好，所以 就它了。

接下来 这里将是我博客的主阵地， 由于时间问题，保证每月至少更新2篇博客。
